I wanted to reflect on how I got my first 100+ stars on a GitHub repository.

## Awesome lists

What are awesome lists?

> An awesome list is a list of awesome things curated by the community. There are awesome lists about everything from CLI applications to fantasy books.

I really like browsing GitHub for 'awesome lists', one of my favourites is [awesome-selfhosted](https://github.com/awesome-selfhosted/awesome-selfhosted) which contains a list of selfhosted tools and resources.

I wanted to create my own collection of free and open-source aviation projects that I know of. While working in aviation, I always had those sources in mind but I never kept a written collection of it, so I am doing this also for my personal benefit.

As of the writing of this post, my list contains almost 20 entries (mostly data sources). In the coming days I intend to expand it further with more data and tools.

## I got spotted

Shortly after publishing the list, I got spotted by [Cyber Detective](https://x.com/cyb_detective), an account which specializes in Open Source Intelligence (OSINT). They shared a link of my repository on X which then got 8k views.

I was quite surprised by the rapidity at which my list (just a simple README file) got discovered. This is probably due to the 'osint' and 'awesome-list' tags that I put on GitHub.

## Growth so far

![GitHub stars chart](https://api.star-history.com/svg?repos=lucianosrp/open-source-aviation&type=Date&theme=dark)

My list was featured in other people's awesome-list which is nice to see. To make it more attractive, I decided to add a Python script that will look for any data source present in that file and append a table containing information for each data source.

I even added a GitHub action to run the script every day, but I recently turned that off as I was not happy with the way it commits the repository.

Overall, I think the key to success was the simplicity of the content as well as a good series of tags to make it more discoverable on GitHub.

## What actually worked

Looking back, the growth wasn't purely luck. Here's what I think contributed:

1. **The OSINT share** — This was the catalyst. But it only worked because the repo was already there. Being discovered is useless if there's nothing worth sharing.

2. **Simplicity** — It's just a README file. No setup, no dependencies, no installation. People can immediately see the value.

3. **Specific niche** — Aviation data is niche but not too narrow. There's demand for curated lists in specialized fields.

4. **Tags** — The osint and awesome-list tags put it in front of the right audience.

5. **First-mover advantage** — At the time, there wasn't a dedicated "awesome-aviation" list. Being early matters.

The takeaway: You can't control virality, but you can control whether your work is worth sharing.
